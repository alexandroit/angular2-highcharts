Angular 10.2 · Highcharts 8

@stackline/angular-highcharts

A thin, unopinionated Angular wrapper for Highcharts, StockChart, 3D charts and heatmaps. Install the package, register the module, and drop <chart> straight into your template.

Zero config Thin wrapper — the full Highcharts API remains directly accessible.
Event bridge Highcharts events become Angular @Output() emitters automatically.
Native instance Capture the Highcharts.Chart object via (load).
Optional modules 3D, heatmap, stock — register only what you need.
See demos README

Setup in 3 steps

1
Install 
npm install @stackline/angular-highcharts@10 highcharts
2
Register in your NgModule 
import { ChartModule } from '@stackline/angular-highcharts';
import * as Highcharts from 'highcharts/highstock';
declare var require: any;
const HC3d   = require('highcharts/highcharts-3d');
const HCHeat = require('highcharts/modules/heatmap');

@NgModule({
  imports: [
    ChartModule.forRoot(Highcharts, HC3d, HCHeat)
  ]
})
3
Use in your template 
<chart [options]="myOptions"></chart>

Core — basic usage

These two demos cover 90 % of everyday use cases.

Basic chart

Pass a standard Highcharts options object to the [options] input.

// component.ts
options = {
  title: { text: 'My chart' },
  series: [{ type: 'line', data: [...] }]
};
<!-- template.html -->
<chart [options]="options"></chart>

StockChart

Set type="StockChart" to use the Highstock constructor.

<chart type="StockChart"
      [options]="stockOptions">
</chart>

Event bridge

Highcharts events are exposed as @Output() on the child components <series>, <point>, <xAxis> and <yAxis>.

Chart, series and point events

The (load) output on <chart> delivers the native instance. (selection) fires when the user drags a zoom area. <series> and <point> forward hover and select events into Angular. 

<chart [options]="eventOptions"
       (load)="onEventLoad($event)"
       (selection)="onSelection($event)">

  <series (mouseOver)="onSeriesHover($event)">
    <point (select)="onPointSelect($event)"></point>
  </series>

  <xAxis (afterSetExtremes)="onXExtremes($event)"></xAxis>
  <yAxis (afterSetExtremes)="onYExtremes($event)"></yAxis>

</chart>

Drag over the chart to select a range, hover over a bar, or click a column. Every event appears in the Event Log on the right.

Native instance access

The wrapper is intentionally thin. For imperative mutations, capture the Highcharts.Chart object from (load) and call the native API directly.

Imperative mutations via the Highcharts API

// component.ts
onDynamicLoad(e: any) {
  this.chart = e.context; // native Highcharts.Chart object
}

addPoint() {
  this.chart.series[0].addPoint(Math.random() * 20, true, false);
}

randomize() {
  this.chart.series[0].setData([...newData], true);
}

renameChart() {
  this.chart.setTitle({ text: 'New title' });
}
<chart [options]="dynamicOptions"
       (load)="onDynamicLoad($event)"></chart>

Extra axis wrappers

<zAxis> and <colorAxis> work the same way as the standard axes: (afterSetExtremes) events and programmatic control via the native instance.

<zAxis> wrapper

3D scatter chart — requires highcharts-3d in forRoot.

<chart [options]="zAxisOptions"
       (load)="onZAxisLoad($event)">
  <zAxis (afterSetExtremes)="onZExtremes($event)">
  </zAxis>
</chart>

<colorAxis> wrapper

Heatmap — requires highcharts/modules/heatmap in forRoot.

<chart [options]="colorAxisOptions"
       (load)="onColorAxisLoad($event)">
  <colorAxis (afterSetExtremes)="onColorExtremes($event)">
  </colorAxis>
</chart>

Optional modules

Pass the modules you need to forRoot. They are applied once at startup. To toggle features at runtime, reassign the options object in your component.

Toggle 3D at runtime

toggle3d() {
  this.enabled = !this.enabled;
  // Reassign the reference so Angular detects the change
  this.moduleOptions = this.makeModuleOptions(this.enabled);
}

Highcharts 6 — new chart types

Angular 10 keeps full support for the Highcharts 6 modules introduced in the previous release lines, including bullet and x-range chart types. Register the modules in forRoot and pass options as usual — no API changes required.

Bullet chart

Compares an actual value to a target and shows qualitative performance bands. New in Highcharts 6.

// app.module.ts
const HighchartsBullet = require('highcharts/modules/bullet');
ChartModule.forRoot(Highcharts, ..., HighchartsBullet)
// component.ts
bulletOptions = {
  chart: { type: 'bullet', inverted: true },
  series: [{ type: 'bullet',
    data: [{ y: 194, target: 200 }, ...] }]
};
<chart [options]="bulletOptions"></chart>

X-Range chart

Horizontal bars spanning a date range — perfect for project timelines. New in Highcharts 6.

// app.module.ts
const HighchartsXRange = require('highcharts/modules/xrange');
ChartModule.forRoot(Highcharts, ..., HighchartsXRange)
// component.ts
xrangeOptions = {
  chart: { type: 'xrange' },
  xAxis: { type: 'datetime' },
  series: [{ type: 'xrange',
    data: [{ x: ..., x2: ..., y: 0 }, ...] }]
};
<chart [options]="xrangeOptions"></chart>

Angular 10 still supports the Highcharts 7 modules, which added relationship and storytelling charts such as timeline, venn, organization, and dependency wheel.

Timeline chart

const HighchartsTimeline = require('highcharts/modules/timeline');
ChartModule.forRoot(Highcharts, ..., HighchartsTimeline)

<chart [options]="timelineOptions"></chart>

Highcharts 8 — new capabilities

Angular 10.2 keeps the Highcharts 8.2 line, bringing newer data interaction patterns such as radial bar, data sorting, and marker clusters.

Radial bar chart

A circular bar layout for compact KPI comparisons. New in the Highcharts 8 line.

// component.ts
radialBarOptions = {
  chart: { polar: true, inverted: true, type: 'column' },
  series: [{ type: 'column', data: [...] }]
};
<chart [options]="radialBarOptions"></chart>

Data sorting

Category series can sort themselves declaratively with animated transitions.

// component.ts
sortedBarOptions = {
  chart: { type: 'bar' },
  series: [{ type: 'bar', dataSorting: { enabled: true }, data: [...] }]
};
<chart [options]="sortedBarOptions"></chart>

Marker clusters

Dense scatter data can be grouped visually with the marker-clusters module.

// app.module.ts
const HighchartsMarkerClusters = require('highcharts/modules/marker-clusters');
ChartModule.forRoot(Highcharts, ..., HighchartsMarkerClusters)
// component.ts
markerClusterOptions = {
  chart: { type: 'scatter' },
  plotOptions: { scatter: { cluster: { enabled: true } } },
  series: [{ type: 'scatter', data: [...] }]
};
<chart [options]="markerClusterOptions"></chart>

Quick API reference

<chart>

Input / OutputTypeDescription
[options]ObjectHighcharts options object (required)
[type]string"Chart" (default) or "StockChart"
(load)ChartEventNative instance available in $event.context
(selection)ChartEventRange selection / zoom drag
(click)ChartEventClick on the chart area
(redraw)ChartEventAfter the chart redraws

<series> / <point>

OutputDescription
(mouseOver)Cursor enters the series
(mouseOut)Cursor leaves the series
(click)Click on a series / point
(select)Point selected
(hide) / (show)Series hidden or shown
(legendItemClick)Legend item clicked

<xAxis> / <yAxis> / <zAxis> / <colorAxis>

OutputDescription
(afterSetExtremes)Extremes changed (zoom / reset)
(setExtremes)Before extremes are applied
(afterBreaks)After axis breaks are applied
(pointBreak)Point falls in a break interval
